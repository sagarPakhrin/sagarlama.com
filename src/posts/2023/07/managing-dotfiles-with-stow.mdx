### What are dotfiles?

Dotfiles are plain text configuration files for most of the applications of unix-like systems. 
These files begin with a dot(`.`) and are hidden by default. The preferences and configurations you'ved made gets saved in the dotfiles.  
If you run `ls -a ~/` you will see a list of dotfiles in your home directory.
In this blog post we will learn how to backup and seamlessly replicate your configurations using git and [GNU Stow](https://www.gnu.org/software/stow/ ).

### Why backup dotfiles?

Wether you want to replicate your configurations on multiple machines, you want to share your config with others, or you just want to backup your config so it doesn't get lost,
no matter the reason, you can still manage your dotfiles with git and stow.

### What is a Symlink
Symbolic Link also known as symlink is simply a shortcut to another file. It is a file that points to another file.

### Install Stow

Stow is going to help us move the files and creating symlinks in the correct paths without having to do it manually.  
Let's start by installing stow

```bash
sudo apt install stow
```


### Setup Directories

Right now, our configurations live all over the place and it's difficult to keep track of it. 
So we will move them to a single folder where we can track it with git.  

```sh
mkdir ~/.dotfiles
cd ~/.dotfiles
git init
```

I like to keep my dotfiles folder hidden so I'll name it as `.dotfiles`.  

We will start with a simple example with begin with. Let's create a directory named `bash` inside our `.dotfiles`

```sh
mkdir bash
cd bash
```

*This `bash` folder will act like the home directory for bash and we need to create the exact folder structure* 
If you run `ls ~/` you will see `.bashrc` right in the home directory. So we will also create a file inside `.dotfiles/bash`

```sh 
touch .bashrc
```

Let's repeate the same steps for gitconfig

```sh
cd ~/.dotfiles
mkdir git
cd git 
touch .gitconfig
```

We created a git folder inside `.dotfiles`, cd into the folder, since the `.gitconfig` was at the home folder, we created `.gitconfig` right inside `git folder`

To see how you can do the same thing for nested folder, let's repeat the same process for `neovim`. 
```sh
cd ~/.dotfiles
mkdir nvim
```

You can call this folder anything you like, `nvim` or `neovim`. The important part is the structure inside this folder.
Neovim config is normally kept inside `~/.config/nvim` folder. If  you run `ls -la ~/.config/nvim` you most likely will have `init.lua` and some other files depending on how you've setup neovim.
Since we will treat the `nvim` folder as the home directory, we will create the `.config/nvim` folder inside `nvim folder`

```sh
mkdir .config
cd .config
mkdir nvim
```
